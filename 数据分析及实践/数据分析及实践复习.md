# 数据分析及实践复习



## 第二章 数据入门

### 一、数据采集

#### 数据采集几种方法：

1.  **检索**：**最灵活、最简单的** 数据获取方式

2. **数据生成**

   - **合成数据**：基于计算机模拟活算法生成模仿现实世界观察的人造数据
     - 优势：大大降低数据获取成本;可控;没有隐私安全问题;数据丰富多样，减少真实世界中存在的偏见

   - **数据增强**：解决目标领域数据少且质量差的问题
     - 合成数据是让模型“无中生有”，直接生成某个领域的数据(有一定质量问题)
       数据增强是让模型“有中生更多”，根据已标注好的数据，举一反三(更安全可靠）

3. **数据众包**：一种利用大众力量来收集、分析和处理数据的模式

4. **批量数据获取：**依靠爬虫程序

1. 三种行为：载入（最复杂）、解析、存储

       利用智能化工具

     - 反爬虫：

       解决方案：
        优化爬虫程序，尽量减少访问次数，尽量不抓取重复内容

3. **解析：**

### 二、数据存储

#### 数据类型与存储方式

1. ##### 结构化数据

   - **定义**：有固定结构，如关系型数据库表。
   - **存储工具**：MySQL、Oracle、SQL Server，支持 SQL 查询，需遵循范式（如第一范式：属性不可再分）。

2. #####  半结构化 / 非结构化 数据

   - **半结构化**：含自描述标记，如 JSON、XML，用 MongoDB（文档型数据库）存储，支持灵活字段。
   - **非结构化**：无固定结构，如图像、文本、视频，可存储为文件或用 HBase（列存储数据库）处理大规模数据。

3. ##### 数据库对比

   - **关系型（SQL）**：适合强一致性场景，如金融系统，但扩展性较差。

   - **非关系型（NoSQL）**：包括键值存储（Redis）、图数据库（Neo4J，适合社交网络），支持高并发和横向扩展。

     | **类型**     | **代表工具** | **优势**                 | **典型场景**         |
     | ------------ | ------------ | ------------------------ | -------------------- |
     | 关系型数据库 | MySQL        | 强一致性、支持复杂查询   | 金融交易、订单管理   |
     | 文档型数据库 | MongoDB      | 灵活 schema、高扩展性    | 日志记录、用户画像   |
     | 键值数据库   | Redis        | 读写速度快、支持缓存     | 实时计数器、会话存储 |
     | 图数据库     | Neo4J        | 高效处理关系数据         | 社交网络、知识图谱   |
     | 列存储数据库 | HBase        | 海量数据存储与快速列查询 | 物联网日志、天文数据 |



### 三、数据预处理



##### 数据质量含义

-  正确性( Correctness)
-  一致性( Consistency) 
- 完整性( Completeness) 
- 可靠性( Reliability)



#### 数据清洗： 提升数据质量

1. ##### 处理缺失值：

   ​	数据缺失的类型
   ​		 完全随机缺失:	不依赖其他属性/变量，不影响样本的无偏性
   ​		 **随机缺失:**		缺失与其他完全属性/变量**有关系** 
   ​		 非随机缺失:	    数据缺失与属性/变量自身的取值有关

   - **删除法**：删除含缺失值的记录或属性，或是改变权重，适用于缺失比例小的场景。

   - **填充法**

     - 统计值填充：均值（适用于数值型）、众数（适用于类别型）、空值填充

     - 模型预测填充：KNN（基于邻近样本）、回归模型（如用年龄预测收入缺失值）。

2. ##### 清洗噪声：

   噪声是测量误差的随机部分（错误值、偏离期望的孤立点值）

   - **分箱平滑**：将数据排序后划分为等深箱，用箱均值 / 中值替代原始值。

     <img src="/Users/halo/Library/Application Support/typora-user-images/image-20250608214223783.png" alt="image-20250608214223783" style="zoom:50%;" />

   - **离群点检测**：通过聚类（如 DBSCAN）识别异常值，手动核查或删除。

   - **回归平滑**：用某些属性预测其他属性，对不符合回归的数据进行平滑处理

1. ##### 冗余检测

   - **属性冗余**：

     1. **Pearson积矩相关系数**, 取值范围为 [-1, 1]

     $$
     r_{A,B} = \frac{\sum (A-\bar{A})(B-\bar{B})}{(n-1)\sigma_A \sigma_B}
     $$
     * 值大于 0, 则属性 **A 和 B 是正相关的, 值越大相关性越强**
     		因此, 表明两个属性中有**一个可以作为冗余删除**
     * 值**等于 0, 则 A 和 B 是独立**的, 它们不存在相关性
     * 值小于 0, 则 A 和 B 是负相关的。

     

     2. **卡方检验**: 值越大, 两个变量相关的可能性越大

     $$
     \chi^2 = \sum_{i=1}^{c} \sum_{j=1}^{r} \frac{(o_{ij} - e_{ij})}{e_{ij}}
     $$
     oij 是联合事件 (Ai; Bj) 的观测频度(即 实际计数)，而 eij 是 (Ai; Bj)的期望频度。 

     卡方检验的原假设是 A 和 B 两个属性相互独立，如果可以 拒绝该原假设，则我们说 A 和 B 是显著相关的

     

   - **样本冗余**：基于欧氏距离或余弦相似度识别重复样本，去重处理。

     1. **欧几里得距离**
        $$
        dist = \sqrt{\sum_{k=1}^{n}(p_k - q_k)^2} \\
        n 表示数据p和q维度数\\
        
        p_k 和 q_k 表示数据 p 和 q 的第 k 个属性
        $$

​		        对数据标准化非常重要

 2. 明氏距离 (Minkowski Distance)
    距离度量：通用表达形式
    $$
    dist = \left( \sum_{k=1}^{n} |p_k - q_k|^r \right)^{\frac{1}{r}}\\
    r 是参数\\
    n 表示数据p和q维度数，p_k 和 q_k 表示数据p和q的第k个属性
    $$



- r=1: 曼哈顿距离

*   r=2: 欧氏距离
*   $ r=  \infty $ : 切比雪夫距离

 3. 马氏距离：数据的协方差距离
                        欧氏距离的扩展，考虑到各种特性之间的联系（协方差）
    $$

    $$
    \Sigma_{j, k}=\frac{1} {n-1} \sum_{i=1}^{n} ( X_{i j}-\overline{{{{X}}}}_{j} ) ( X_{i k}-\overline{{{{X}}}}_{k} )

#### 数据规约：缩小数据挖掘所需要的数据集规模

1. **维度规约**：删除与挖掘任务不相关的属性

   - **主成分分析（PCA）**

     把高维（N）数据压缩为低维（K）向量，这样K维空间每一个基向量就是一个主成分

     - <u>消除原始数据不同属性间的相关性</u>，要求:K个维度间相互独立

     - <u>最大化保留K维度上的数据多样性</u>，要求:最大化每个维度内的样本方差。

       ​	使用方差信息 ，若在一个方向上发现数据分布的方差越大，则说明该投影方向越能体现数据中的主要信息。该投影方向即应当是一个主成分

   - **特征子集选择**

     从原始属性中挑选最相关的子集，剔除冗余或无关属性。

2. **数值规约**：用较小的数据表示形式替换原始数据（ex.用模型替换）





### 四、特征工程

#### 特征设计

##### **一、基础特征提取**

**数据类型转换与预处理**

1. 数值型特征
   - 归一化：缩放到指定区间（如 [0,1]）
   - 标准化：转化为均值 0、标准差 1 的分布
2. 类别型特征
   - **独热编码（One-hot）**：将类别映射为二进制向量（如 “性别”→[1,0]/[0,1]）
     - 无法体现词语语义关联（如 “函数” 与 “偶函数”）。
     - 存在维度灾难（如 100 个类别→100 维）。
   - **标签编码（Label Encoding）**：用数字编码类别（如 “低 = 1，中 = 2，高 = 3”），适用于线性模型。

**缺失值与噪声处理**

- 缺失值填充：
  - 统计值填充（均值 / 中位数 / 众数）、KNN 近邻填充、回归模型预测填充。
- 噪声平滑：
  - 分箱法（按箱均值 / 中值平滑）、回归分析拟合趋势。

##### 二、创建新特征

**领域知识驱动的特征构造**

- 比值与交叉特征：
  - *案例*：电商场景中 “收藏次数 / 加购次数” 反映购买意向，“浏览时长 × 商品价格” 反映潜在消费力。
- 时序特征：
  - 滑动窗口统计（均值、最大值、标准差）、趋势差分（如一阶差分反映变化率）。

**函数变换特征**

- 数学变换：
  - 对数变换（处理右偏分布，如房价）、平方根变换、指数变换。
- 傅里叶变换：
  - *案例*：将含噪声的正弦波时间序列转换为频率图，提取主要频率成分（如 7Hz 和 17Hz）。

##### **三、文本特征设计**

**词频统计与词袋模型（BOW）**

- **核心思想**：忽略词序，统计词语出现频率。

- 案例：

  - 文本 “John likes to watch movies. Mary likes too.” 的词频向量：likes=2,John=1,Mary=1,watch=1,movies=1,too=1

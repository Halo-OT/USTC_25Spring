### 第一部分：SQL概述、数据定义与数据更新

#### 1. SQL概述
- **SQL（Structured Query Language）**：结构化查询语言，是关系数据库的标准语言，功能强大，高度非过程化，支持集合操作，语言简洁易学。
- **SQL的特点**：
  1. **综合统一**：集数据定义语言（DDL）、数据操纵语言（DML）、数据控制语言（DCL）功能于一体，支持数据库生命周期中的全部活动。
  2. **高度非过程化**：用户只需说明“做什么”，无须了解存取路径，存取路径的选择由系统自动完成。
  3. **面向集合的操作方式**：操作对象和查找结果可以是元组的集合，一次插入、删除、更新操作的对象可以是元组的集合。
  4. **同一种语法结构，多种使用方式**：SQL可以独立使用，也可以嵌入到高级语言（如C、C++、Java）程序中。
  5. **语言简洁，易学易用**：完成核心功能只用了9个动词（CREATE, DROP, ALTER, SELECT, INSERT, UPDATE, DELETE, GRANT, REVOKE）。
  
- **SQL的基本概念**：
  - **三级模式结构**：
    - **外模式**：视图（View）和部分基本表（Base table）。
    - **模式**：若干基本表（Base table）。
    - **内模式**：存储文件（stored file）。
  - **基本表**：本身独立存在的表，一个关系对应一个基本表，一个或多个基本表对应一个存储文件，一个表可以带若干索引。
  - **存储文件**：逻辑结构组成了关系数据库的内模式，物理结构对用户透明。
  - **视图**：从一个或几个基本表导出的表，数据库中只存放视图的定义，不存放视图对应的数据，是一个虚表。

#### 2. 数据定义
- **模式的定义与删除**：
  - **创建模式**：`CREATE SCHEMA <模式名> AUTHORIZATION <用户名>`。
    - 用户名是已存在的数据库用户
    - 模式名default：用户名
  - **删除模式**：`DROP SCHEMA <模式名> CASCADE|RESTRICT`。
    - **CASCADE**：删除模式的同时，删除该模式中所有的数据库对象。
    - **RESTRICT**：如果模式中定义了下属的数据库对象，则拒绝删除。
  
- **基本表的定义、删除与修改**：
  - **创建表**：
    ```sql
    CREATE TABLE <表名> (
        <列名> <数据类型> [列级完整性约束条件],
        ...
        [表级完整性约束条件]
    );
    ```
    - **示例**：创建“学生”表`Student`，学号是主码，姓名取值唯一。
      ```sql
      CREATE TABLE Student (
          Sno CHAR(9) PRIMARY KEY,
          Sname CHAR(20) UNIQUE,
          Ssex CHAR(2),
          Sage SMALLINT,
          Sdept CHAR(20)
      );
      
      CREATE TABLE SC( 
        Sno CHAR(9),
        Cno CHAR(4),
        Grade SMALLINT, 
        Semester CHAR(5), 
        PRIMARY KEY (Sno，Cno),
        	/* 主码由两个属性构成，必须作为表级完整性进行定义*/ 
        FOREIGN KEY (Sno) REFERENCES Student(Sno),
        	/* 表级完整性约束条件，Sno是外码，被参照表是Student */ 
        FOREIGN KEY (Cno) REFERENCES Course(Cno)
        	/* 表级完整性约束条件， Cno是外码，被参照表是Course*/
      );
      ```
  - **删除表**：`DROP TABLE <表名> CASCADE|RESTRICT`。
    - **CASCADE**：删除表的同时，删除相关的依赖对象（如索引、视图等）。
    - **RESTRICT**：如果表上有依赖对象，则拒绝删除。
  - **修改表**：
    ```sql
    ALTER TABLE <表名>
    [ADD[COLUMN] <新列名> <数据类型> [完整性约束]]
    [DROP[COLUMN] <列名> [CASCADE|RESTRICT]]
    [ALTER COLUMN <列名> <数据类型>];
    ```
    - **示例**：向`Student`表增加“邮箱地址”列`Semail`。
      ```sql
      ALTER TABLE Student ADD Semail VARCHAR(30);
      ```
  
- **索引的建立与删除**：
  - **建立索引**：
    ```sql
    CREATE [UNIQUE] [CLUSTER] INDEX <索引名> ON <表名>(<列名>[<次序>][,<列名>[<次序>]...]);
    ```
    - **示例**：在`Student`表的`Sage`列上建立一个聚簇索引。
      ```sql
      CREATE CLUSTER INDEX Stuage ON Student(Sage);
      ```
  - **删除索引**：`DROP INDEX <索引名>`。
    - **示例**：删除`Student`表的`Stusname`索引。
      ```sql
      DROP INDEX Stusname;
      ```

- **数据字典**：
  - 数据字典是关系数据库管理系统内部的一组系统表，记录了数据库中所有定义信息，包括关系模式定义、视图定义、索引定义、完整性约束定义、用户操作权限、统计信息等。
  - RDBMS在执行SQL的数据定义语句时，实际上是在更新数据字典表中的相应信息。

#### 3. 数据更新
- **插入数据**：
  - **插入元组**：
    ```sql
    INSERT INTO <表名> [(<属性列1>[,<属性列2>...))] VALUES (<常量1>[,<常量2>...]);
    ```
    - **示例**：将一个新学生元组（学号：20180009；姓名：陈冬；性别：男；所在系：IS；年龄：18岁）插入到`Student`表中。
      ```sql
      INSERT INTO Student (Sno, Sname, Ssex, Sdept, Sage) 
      VALUES ('20180009', '陈冬', '男', 'IS', 18);
      ```
  - **插入子查询结果**：
    
    ```sql
    INSERT INTO <表名> [(<属性列1>[,<属性列2>...))] SELECT <目标列表达式> FROM <表名>;
    ```
    - **示例**：对每一个系，求学生的平均年龄，并把结果存入数据库。
      ```sql
      CREATE TABLE Dept_age (Sdept CHAR(15), Avg_age SMALLINT);
      INSERT INTO Dept_age (Sdept, Avg_age) 
        SELECT Sdept, AVG(Sage) 
        FROM Student 
        GROUP BY Sdept;
      ```
  
- **修改数据**：
  - **修改语句**：
    ```sql
    UPDATE <表名> SET <列名>=<表达式>[,<列名>=<表达式>...] [WHERE <条件>];
    ```
    - **示例**：将学生20180001的年龄改为18岁。
      
      ```sql
      UPDATE Student 
      SET Sage=18 
      WHERE Sno='20180001';
      /*省略 WHERE 将意味着修改表中所有的元组*/
      
      /*带子查询的修改语句*/
      UPDATE SC
      SET Grade=0 
      WHERE Sno IN 
        (SELECT Sno
        FROM Student
        WHERE Sdept = ‘CS’);
      ```
  
- **删除数据**：
  - **删除语句**：
    ```sql
    DELETE FROM <表名> [WHERE <条件>];
    ```
    - **示例**：删除学号为20180008的学生记录。
      ```sql
      DELETE FROM Student 
      WHERE Sno='20180008';
      /*缺少 where 代表删除表中的全部元组，但表的定义仍在字典中*/
      ```

- 注意数据更新中，可能由于sql的self-updates模式使得非主键条件下的update或delete指令失效，

  修改方式：SET SQL_SAFE_UPDATES = 0;

### 第二部分：数据查询

#### 1. 数据查询概述
- **SELECT语句的一般格式**：
  ```sql
  SELECT [ALL|DISTINCT] <目标列表达式>[,<目标列表达式>] …
  FROM <表名或视图名>[,<表名或视图名> ]…|(SELECT 语句)
  [AS]<别名>
  [WHERE <条件表达式>]
  [GROUP BY <列名1> [HAVING <条件表达式>]]
  [ORDER BY <列名2> [ASC|DESC]];
  ```
  - **SELECT子句**：指定要显示的属性列。
  - **FROM子句**：指定查询对象（基本表或视图）。
  - **WHERE子句**：指定查询条件。
  - **GROUP BY子句**：对查询结果按指定列的值分组。
  - **HAVING短语**：只有满足指定条件的组才予以输出。
  - **ORDER BY子句**：对查询结果表按指定列值的升序或降序排序。

#### 3. 连接查询

- **等值与非等值连接查询**：
  - 等值连接：
    ```sql
    SELECT Student.*, SC.* 
    FROM Student, SC 
    WHERE Student.Sno = SC.Sno;
    ```
    
    - 连接操作的执行过程
    
      ##### 嵌套循环法(NESTED-LOOP)
      - **操作步骤**：
          - 首先在表1中找到第一个元组，然后从头开始扫描表2，逐一查找满足连接条件的元组，找到后就将表1中的第一个元组与该元组拼接起来，形成结果表中一个元组。
          - 表2全部查找完后，再找表1中第二个元组，然后再从头开始扫描表2，逐一查找满足连接条件的元组，找到后就将表1中的第二个元组与该元组拼接起来，形成结果表中一个元组。
          - 重复上述操作，直到表1中的全部元组都处理完毕。
    
      ##### 排序合并法
      - **操作步骤**：
          - 首先按连接属性对表1和表2排序。
          - 对表1的第一个元组，从头开始扫描表2，顺序查找满足连接条件的元组，找到后就将表1中的第一个元组与该元组拼接起来，形成结果表中一个元组。当遇到表2中第一条大于表1连接字段值的元组时，对表2的查询不再继续。
          - 找到表1的第二条元组，然后从刚才的中断点处继续顺序扫描表2，查找满足连接条件的元组，找到后就将表1中的第一个元组与该元组拼接起来，形成结果表中一个元组。直接遇到表2中大于表1连接字段值的元组时，对表2的查询不再继续。
          - 重复上述操作，直到表1或表2中的全部元组都处理完毕为止。
    
      ##### 索引连接(INDEX-JOIN)
      - **操作步骤**：
          - 对表2按连接字段建立索引。
          - 对表1中的每个元组，依次根据其连接字段值查询表2的索引，从中找到满足条件的元组，找到后就将表1中的第一个元组与该元组拼接起来，形成结果表中一个元组。
  - 自然连接：
  
    ```sql
    SELECT Student.Sno, Sname, Ssex, Sage, Sdept, Cno, Grade 
    FROM Student, SC 
    WHERE Student.Sno = SC.Sno;
    ```
  
- **自身连接**：
  - 一个表与其自己进行连接：
    
    需要给表起别名展示区别
    
    ```sql
    查找一门课（Cno）先修课（Cpno）的先修课
    SELECT FIRST.Cno, SECOND.Cpno 
    FROM Course FIRST, Course SECOND 
    WHERE FIRST.Cpno = SECOND.Cno 
    			AND SECOND.Cpno IS NOT NULL;
    ```
  
- **外连接**：
  - 左外连接：
    ```sql
    SELECT Student.Sno, Sname, Ssex, Sage, Sdept, Cno, Grade 
    FROM Student 
    LEFT OUTER JOIN SC ON (Student.Sno = SC.Sno);
    ```
  
- **多表连接**：
  - 两个以上的表进行连接：
    ```sql
    SELECT Student.Sno, Sname, Cname, Grade 
    FROM Student, SC, Course 
    WHERE Student.Sno = SC.Sno AND SC.Cno = Course.Cno;
    ```

#### 4. 嵌套查询

- 子查询限制：不能使用 ORDER BY字句

  有的嵌套查询可转用连接运算替代

  不相关子查询 def：子查询的查询条件不依赖父查询 —— 可转为连接查询

- **带有IN谓词的子查询**：
  - 查询与“刘晨”在同一个系学习的学生：
    ```sql
    SELECT Sno, Sname, Sdept 
    FROM Student 
    WHERE Sdept IN 
    						(SELECT Sdept 
                 FROM Student 
                 WHERE Sname = '刘晨');
                 
    上为不相关子查询，可转为：
    SELECT S1.Sno，S1.Sname，S1.Sdept
    FROM Student S1，Student S2
    WHERE S1.Sdept = S2.Sdept AND S2.Sname = '刘晨'
    ```
  
- **带有比较运算符的子查询**：

  - 找出每个学生超过他选修课程平均成绩的课程号
    ```sql
    SELECT Sno， Cno
    FROM SC x
    WHERE Grade >= (SELECT AVG(Grade)
                    FROM SC y
                    WHERE y.Sno = x.Sno);
                    
    此为相关子查询
    ```

- **带有ANY（SOME）或ALL谓词的子查询**：
  - 查询非计算机系中比计算机系任意一个学生年龄小的学生姓名，年龄和系名：
    ```sql
    SELECT Sname, Sage, Sdept FROM Student 
    WHERE Sage < ANY (SELECT Sage 
                      FROM Student 
                      WHERE Sdept = 'CS') 
          AND Sdept <> 'CS';
    ```

- **带有EXISTS谓词的子查询**：

  存在于/不存在于 某张子查询表中

  - 查询所有选修了1号课程的学生姓名：
    ```sql
    SELECT Sname 
    FROM Student 
    WHERE EXISTS (SELECT * 
                  FROM SC 
                  WHERE Sno = Student.Sno AND Cno = '1');
                  
    这个查询检查每个学生是否在SC表中有选修1号课程的记录，如果有，则将该学生的姓名加入结果。
    ```


- 不同形式的查询间的替换
  - 一些带EXISTS或NOT EXISTS谓词的子查询不能被其 他形式的子查询等价替换
  - 所有带IN谓词、比较运算符、ANY和ALL谓词的子查 询都能用带EXISTS谓词的子查询等价替换

### 第三部分：集合查询、基于派生表的查询、空值处理与视图

#### 1. 集合查询
- **集合操作的种类**：
  - **并操作（UNION）**：将多个查询结果合并，自动去掉重复元组。
    
    UNION ALL 保留重复元组
    
    ```sql
    SELECT * 
    FROM Student 
    WHERE Sdept = 'CS' 
    UNION 
    SELECT * 
    FROM Student 
    WHERE Sage <= 19;
    ```
    
  - **交操作（INTERSECT）**：返回多个查询结果的交集。
    ```sql
    SELECT * FROM Student 
    WHERE Sdept = 'CS' 
    INTERSECT 
    SELECT * FROM Student 
    WHERE Sage <= 19;
    ```
    
  - **差操作（EXCEPT）**：返回第一个查询结果中不在第二个查询结果中的元组。
    ```sql
    SELECT * FROM Student WHERE Sdept = 'CS' 
    EXCEPT 
    SELECT * FROM Student WHERE Sage <= 19;
    ```
  
- **注意事项**：
  - 参加集合操作的各查询结果的列数必须相同，对应项的数据类型也必须相同。

#### 2. 基于派生表的查询
- **派生表**：子查询出现在`FROM`子句中，生成的临时派生表成为主查询的查询对象。
  - **示例**：找出每个学生超过他自己选修课程平均成绩的课程号。
    ```sql
    SELECT Sno, Cno
    FROM SC x
    WHERE Grade >= (SELECT AVG(Grade) 
                    FROM SC y 
                    WHERE y.Sno = x.Sno);
    ```
  - **另一种方法**：
    ```sql
    SELECT Sno, Cno
    FROM SC, (SELECT Sno, AVG(Grade) AS AvgGrade 
              FROM SC 
              GROUP BY Sno) AS Avg_sc
    WHERE SC.Sno = Avg_sc.Sno AND SC.Grade >= Avg_sc.AvgGrade;
    ```

#### 3. 空值的处理
- **空值的产生**：
  - 插入空值：
    ```sql
    INSERT INTO SC(Sno, Cno, Grade, Semester) 
    VALUES('20180006', '4', NULL, '20211');
    ```
  - 更新为NULL：
    ```sql
    UPDATE Student 
    SET Sdept = NULL 
    WHERE Sno = '20180006';
    ```
  
- **空值的判断**：
  - 使用`IS NULL`或`IS NOT NULL`：
    ```sql
    SELECT * 
    FROM Student 
    WHERE Sname IS NULL OR Ssex IS NULL OR Sage IS NULL OR Sdept IS NULL;
    ```
  
- **空值的约束条件**：
  - `NOT NULL`约束的列不能取空值。
  - `UNIQUE`限制的列不能取空值。
  - 主码列不能取空值。

- **空值的运算**：
  - 空值与另一个值的算术运算结果为空值。
  - 空值与另一个值的比较运算结果为UNKNOWN。
  - 三值逻辑（TRUE、FALSE、UNKNOWN）：
    ```sql
    SELECT Sno FROM SC WHERE Grade < 60 AND Cno = '1';
    SELECT Sno FROM SC WHERE Grade IS NULL AND Cno = '1';
    ```

#### 4. 视图
- **视图的特点**：
  - 虚表，从一个或多个基本表导出。
  - 只存放视图的定义，不存放视图对应的数据。
  - 基表中的数据发生变化，视图中的数据也随之改变。

- **基于视图的操作**：
  - 查询视图：
    ```sql
    SELECT Sno, Sage FROM IS_Student WHERE Sage < 20;
    ```
  - 更新视图（受限）：
    ```sql
    UPDATE IS_Student SET Sname = '刘新奇' WHERE Sno = '20180005';
    ```

- **定义视图**：
  - 创建视图：
    ```sql
    CREATE VIEW IS_Student 
    AS 
      SELECT Sno, Sname, Ssex, Sage, Sdept 
      FROM Student 
      WHERE Sdept = 'IS';
      
    CREATE VIEW IS_C1(Sno，Sname，Grade，Semester)
    AS
    SELECT Student.Sno，Sname，Grade，Semester 
    FROM Student，SC
    WHERE Sdept= 'IS' AND 
    			Student.Sno=SC.Sno AND
          SC.Cno= ' 1 ';
    ```
  - 带`WITH CHECK OPTION`的视图：
    ```sql
    CREATE VIEW IS_Student 
    AS 
      SELECT Sno, Sname, Ssex, Sage, Sdept 
      FROM Student 
      WHERE Sdept = 'IS' 
    WITH CHECK OPTION;
    ```

- **删除视图**：
  - 删除视图：
    ```sql
    DROP VIEW IS_Student;
    ```

- ##### 查询视图

  ##### 1. 查询视图的基本概念
  - **用户角度**：从用户的角度来看，查询视图与查询基本表的操作完全相同。用户可以像查询普通表一样查询视图。
  - **RDBMS实现**：RDBMS在实现视图查询时，通常采用视图消解法（View Resolution）。具体步骤如下：
    1. **有效性检查**：检查视图定义是否有效，确保视图的定义与当前数据库状态一致。
    2. **转换为对基本表的查询**：将对视图的查询转换为对基本表的等价查询。
    3. **执行修正后的查询**：执行转换后的查询语句，返回结果。

  ##### 2. 查询视图的示例
  - **示例1**：在信息系学生的视图中找出年龄小于20岁的学生。
    ```sql
    SELECT Sno, Sage FROM IS_Student WHERE Sage < 20;
    ```
    - **视图定义**：
      ```sql
      CREATE VIEW IS_Student AS
      SELECT Sno, Sname, Ssex, Sage, Sdept
      FROM Student
      WHERE Sdept = 'IS';
      ```
    - **视图消解后的查询**：
      ```sql
      SELECT Sno, Sage
      FROM Student
      WHERE Sdept = 'IS' AND Sage < 20;
      ```

  - **示例2**：查询选修了1号课程的信息系学生。
    ```sql
    SELECT IS_Student.Sno, Sname
    FROM IS_Student, SC
    WHERE IS_Student.Sno = SC.Sno AND SC.Cno = '1';
    ```
    - **视图消解后的查询**：
      ```sql
      SELECT Student.Sno, Sname
      FROM Student, SC
      WHERE Student.Sno = SC.Sno AND SC.Cno = '1' AND Student.Sdept = 'IS';
      ```

  - **示例3**：在S_GradeAVG视图中查询平均成绩在90分以上的学生学号和平均成绩。
    ```sql
    SELECT * FROM S_GradeAVG WHERE Gavg >= 90;
    ```
    - **视图定义**：
      ```sql
      CREATE VIEW S_GradeAVG (Sno, Gavg) AS
      SELECT Sno, AVG(Grade)
      FROM SC
      GROUP BY Sno;
      ```
    - **视图消解后的查询**：
      ```sql
      SELECT Sno, AVG(Grade)
      FROM SC
      GROUP BY Sno
      HAVING AVG(Grade) >= 90;
      ```

  ##### 3. 视图消解法的局限
  - **复杂查询**：在某些情况下，视图消解法可能不适用或需要特别注意。例如，当视图定义中包含复杂的子查询或聚合函数时，直接转换可能会导致错误或不等价的查询。
  - **示例**：在S_GradeAVG视图中查询平均成绩在90分以上的学生学号和平均成绩。
    - **错误的转换**：
      ```sql
      SELECT Sno, AVG(Grade)
      FROM SC
      WHERE AVG(Grade) >= 90
      GROUP BY Sno;
      ```
    - **正确的转换**：
      ```sql
      SELECT Sno, AVG(Grade)
      FROM SC
      GROUP BY Sno
      HAVING AVG(Grade) >= 90;
      ```

- #####  更新视图

  更新可更新视图会同步更新基本表

  ##### 1. 更新视图的基本概念
  - **WITH CHECK OPTION**：在定义视图时，可以使用`WITH CHECK OPTION`来确保对视图进行`UPDATE`、`INSERT`和`DELETE`操作时，更新、插入或删除的行满足视图定义中的谓词条件。
  - **更新操作**：
    - **修改操作**：自动加上视图定义中的条件。
    - **删除操作**：自动加上视图定义中的条件。
    - **插入操作**：自动检查是否满足视图定义中的条件。如果不满足，则拒绝该插入操作。如果没有提供某些属性值，则自动定义这些属性值。

  ##### 2. 更新视图的示例
  - **示例1**：将信息系学生视图IS_Student中学号为20180005的学生姓名改为“刘新奇”。
    ```sql
    UPDATE IS_Student
    SET Sname = '刘新奇'
    WHERE Sno = '20180005';
    ```
    - **转换后的语句**：
      ```sql
      UPDATE Student
      SET Sname = '刘新奇'
      WHERE Sno = '20180005' AND Sdept = 'IS';
      ```

  - **示例2**：向信息系学生视图IS_Student中插入一个新的学生记录：20180207，赵新，男，20岁，IS系。
    ```sql
    INSERT INTO IS_Student VALUES ('20180207', '赵新', '男', 20, 'IS');
    ```
    - **转换为对基本表的更新**：
      ```sql
      INSERT INTO Student (Sno, Sname, Ssex, Sage, Sdept)
      VALUES ('20180207', '赵新', '男', 20, 'IS');
      ```

  - **示例3**：删除信息系学生视图IS_Student中学号为20180207的记录。
    ```sql
    DELETE FROM IS_Student
    WHERE Sno = '20180207';
    ```
    - **转换为对基本表的更新**：
      ```sql
      DELETE FROM Student
      WHERE Sno = '20180207' AND Sdept = 'IS';
      ```

  ##### 3. 更新视图的限制
  - **不可更新的视图**：一些视图是不可更新的，因为对这些视图的更新不能唯一地有意义地转换成对相应基本表的更新。例如：
    - 视图定义中包含聚合函数。
    - 视图定义中包含`GROUP BY`子句。
    - 视图定义中包含`DISTINCT`短语。
    - 视图定义中包含嵌套查询，且内层查询的`FROM`子句中涉及的表也是导出该视图的基本表。
    - 视图定义中包含字段表达式或常数。
    - 一个不允许更新的视图上定义的视图也不允许更新。

  - **示例**：更新S_GradeAVG视图中某个学生的平均成绩。
    ```sql
    UPDATE S_GradeAVG
    SET Gavg = 90
    WHERE Sno = '20180001';
    ```
    - **问题**：这个对视图的更新无法转换成对基本表SC的更新，因为S_GradeAVG视图是基于聚合函数定义的。

  ##### 4. 允许更新的视图
  - **行列子集视图**：如果视图是从单个基本表导出的，并且只是去掉了基本表的某些行和某些列，但保留了主码，这类视图称为行列子集视图。行列子集视图是可以更新的。

- **视图的作用**：

  - 简化用户操作。
  - 使用户能以多种角度看待同一数据。
  - 提供一定程度的逻辑独立性。
  - 对机密数据提供安全保护。
  - 适当的利用视图可以更清晰的表达查询。

# 数据库完整性

### 1. 数据库完整性概述
数据库完整性是指数据的**正确性**和**相容性**，确保数据符合现实世界的语义，并在不同关系表中保持逻辑一致。完整性与安全性不同，安全性关注的是防止非法访问和破坏，而完整性关注的是防止数据不正确。

为维护数据库的完整性，DBMS需要提供以下机制：
- **定义完整性约束条件**：通过SQL的DDL语句实现。
- **完整性检查**：在数据操作（如INSERT、UPDATE、DELETE）后或事务提交时进行。
- **违约处理**：如拒绝操作或级联操作。

### 2. 实体完整性
实体完整性要求关系表的**主属性（主键）不能取空值**，确保每个实体（记录）是唯一的。主键可以通过以下方式定义：
- **列级约束**：在列定义时直接指定`PRIMARY KEY`。
- **表级约束**：在表定义的最后指定`PRIMARY KEY`。

实体完整性的违约处理：

- 拒绝插入或修改
- 检查主码值是否唯一的一种方法：全表扫描

例如：
```sql
CREATE TABLE Student (
    Sno CHAR(9) PRIMARY KEY,
    Sname CHAR(20) NOT NULL,
    Ssex CHAR(2),
    Sage SMALLINT,
    Sdept CHAR(20)
);
```

### 3. 参照完整性
参照完整性确保**外键值必须匹配被参照表的主键值，或者为空**。通过`FOREIGN KEY`和`REFERENCES`定义外键关系。参照完整性的违约处理方式包括：
- **拒绝操作**（NO ACTION）：默认策略。

- **级联操作**（CASCADE）：自动删除或更新相关记录。

- **设置为空值**（SET NULL）：将相关记录的外键值设置为空。

### 4. 用户定义的完整性
用户定义的完整性是针对具体应用的语义要求，通过SQL的`CHECK`约束实现。可以定义属性上的约束（如非空、唯一）或元组上的约束（如跨多个属性的条件）。

例如：
```sql
CREATE TABLE Student (
    Sno CHAR(9) PRIMARY KEY,
    Sname CHAR(20) NOT NULL,
    Ssex CHAR(2) CHECK (Ssex IN ('男', '女')),
    Sage SMALLINT CHECK (Sage >= 14 AND Sage <= 50),
    Sdept CHAR(20)
);
```

### 5. 完整性约束命名子句
完整性约束可以通过`CONSTRAINT`命名，方便管理和修改。例如：
```sql
CREATE TABLE Student (
    Sno NUMERIC(6) CONSTRAINT C1 CHECK (Sno BETWEEN 90000 AND 99999),
    Sname CHAR(20) CONSTRAINT C2 NOT NULL,
    Sage NUMERIC(3) CONSTRAINT C3 CHECK (Sage < 30),
    Ssex CHAR(2) CONSTRAINT C4 CHECK (Ssex IN ('男', '女')),
    CONSTRAINT StudentKey PRIMARY KEY (Sno)
);


可以先删除原来的约束条件，再增加新的约束条件 
ALTER TABLE Student
DROP CONSTRAINT C1; 

ALTER TABLE Student
ADD CONSTRAINT C1 CHECK (Sno BETWEEN 900000 AND 999999); 

ALTER TABLE Student
DROP CONSTRAINT C3;

ALTER TABLE Student
ADD CONSTRAINT C3 CHECK (Sage < 40);
```

### 6. 触发器
触发器是用户定义在关系表上的一类由事件驱动的特殊过程，用于实施更复杂的检查和操作。触发器可以定义在INSERT、UPDATE或DELETE操作之前或之后执行。

#### 6.1 定义触发器
触发器的语法格式如下：
```sql
CREATE TRIGGER <触发器名>
{BEFORE | AFTER} <触发事件> ON <表名>
REFERENCING NEW|OLD ROW AS <变量>
FOR EACH {ROW | STATEMENT}
[WHEN <触发条件>] <触发动作体>;
```

例如：

- **记录修改操作**：
  ```sql
  CREATE TRIGGER SC_T
  AFTER UPDATE OF Grade ON SC
  REFERENCING OLD ROW AS OldTuple, NEW ROW AS NewTuple
  FOR EACH ROW
  WHEN (NewTuple.Grade >= 1.1 * OldTuple.Grade)
  INSERT INTO SC_U (Sno, Cno, OldGrade, NewGrade)
  VALUES (OldTuple.Sno, OldTuple.Cno, OldTuple.Grade, NewTuple.Grade);
  ```

- **记录插入操作**：
  ```sql
  CREATE TRIGGER Student_Count
  AFTER INSERT ON Student
  REFERENCING NEW TABLE AS DELTA
  FOR EACH STATEMENT
  INSERT INTO StudentInsertLog (Numbers)
  SELECT COUNT(*) 
  FROM DELTA;
  ```
  
- **自动修改数据**：
  
  ```sql
  CREATE TRIGGER Insert_Or_Update_Sal
  BEFORE INSERT OR UPDATE ON Teacher
  REFERENCING NEW ROW AS NewTuple
  FOR EACH ROW
  BEGIN
      IF (NewTuple.Job = '教授' AND NewTuple.Sal < 4000) THEN
          NewTuple.Sal := 4000;
      END IF;
  END;
  ```

#### 6.2 激活触发器
触发器由数据库服务器自动激活，执行顺序为：
1. 执行`BEFORE`触发器。
2. 执行触发事件的SQL语句。
3. 执行`AFTER`触发器。

#### 6.3 删除触发器
删除触发器的语法如下：
```sql
DROP TRIGGER <触发器名> ON <表名>;
```

### 7. 小结
数据库完整性是确保数据正确性的关键，通过完整性约束定义、检查机制和违约处理机制实现。触发器提供了更灵活和强大的数据控制能力，可以用于实现复杂的完整性规则。

### 第四章：数据库安全性

#### 1. 数据库安全性概述
- **数据库不安全因素**：
  - **非授权用户对数据库的恶意存取和破坏**：黑客通过获取用户名和口令，假冒合法用户进行非法操作。
  
    <u>安全措施包括：用户身份鉴别、存取控制、视图等</u>

  - **数据库中重要或敏感的数据被泄露**：黑客和敌对分子盗窃重要数据，导致机密信息泄露。
  
    <u>安全措施包括：强制存取控制、数据加密存储和加密传输</u>
  
    <u>审计日志分析</u>
  
  - **安全环境的脆弱性**：计算机硬件、操作系统、网络系统的安全性问题。
  
- **安全标准简介**：
  - **TCSEC（可信计算机系统评估准则）**：1985年美国国防部颁布，分为不同安全级别（如C1、C2、B1、B2、A1）。
  - **CC（Common Criteria）**：国际标准，2001年被我国采用，分为不同评估保证级（EAL1到EAL7）。

#### 2. 数据库安全性控制
- **用户身份鉴定**：
  - **静态口令鉴别**：用户设定的静态不变的口令。
  - **动态口令鉴别**：口令动态变化，每次登录使用新口令。
  - **生物特征鉴别**：通过指纹、虹膜等生物特征进行认证。
  - **智能卡鉴别**：使用内置集成电路芯片的智能卡进行硬件加密。

- **存取控制**：
  - **自主存取控制（DAC）**：用户自主决定数据的存取权限，灵活性高（C2级）。
  - **强制存取控制（MAC）**：系统强制控制，数据和用户标记安全等级，严格性高（B1级）。

- **授权与收回**：
  - **GRANT语句**：授予用户权限。
    ```sql
    GRANT SELECT, UPDATE 
    ON TABLE Student 
    TO U1;
    
    GRANT ALL PRIVILIGES
    ON TABLE Student, Course
    TO U2, U3;
    
    GRANT SELECT 
    ON TABLE SC
    TO PUBLIC;
    
    GRANT UPDATE(Sno), SELECT 
    ON TABLE Student
    TO U4;
    
    GRANT INSERT
    ON TABLE SC
    TO U5
    WITH GRANT OPTION;
    /*WITH GRANT OPTION 有该子句可以再授予权限给别人，没指定则不能再传播*/
    ```
  - **REVOKE语句**：收回用户权限。
    ```sql
    REVOKE SELECT 
    ON TABLE Student 
    FROM U1;
    
    REVOKE SELECT
    ON TABLE SC 
    FROM PUBLIC;
    
    REVOKE INSERT
    ON TABLE SC
    FROM U5 CASCADE ;
    /*级联收回别人从U5处获得的权限*/
    ```
  - **角色管理**：通过角色简化授权过程。
    ```sql
    CREATE ROLE R1;
    GRANT SELECT, UPDATE, INSERT ON TABLE Student TO R1;
    
    GRANT R1 TO 王平, 张明;
    [WITH ADMIN OPTION]/*可以把该角色权限授予别人*/
    
    REVOKE R1
    FROM 王平
    ```
  
- **数据库角色**：
  
  - **创建角色**：`CREATE ROLE <角色名>`。
  - **给角色授权**：`GRANT <权限> ON <对象> TO <角色>`。
  - **将角色授予用户**：`GRANT <角色> TO <用户>`。
  - **收回角色权限**：`REVOKE <权限> ON <对象> FROM <角色>`。
  
- **强制存取控制方法**：
  - **主体和客体的敏感度标记**：
    - 主体：许可证级别（如绝密、机密、可信、公开）。
    - 客体：密级（如绝密、机密、可信、公开）。
  - **存取控制规则**：
    - 读取：主体许可证级别 ≥ 客体密级。
    - 写入：主体许可证级别 ≤ 客体密级。

#### 3. 视图机制
- **视图机制**：
  - 通过视图隐藏敏感数据，提供一定程度的安全保护。
  - 视图与授权机制配合使用。
  - 示例：建立计算机系学生的视图并授权。
    ```sql
    CREATE VIEW CS_Student AS SELECT * FROM Student WHERE Sdept = 'CS' WITH CHECK OPTION;
    GRANT SELECT ON CS_Student TO 王平;
    ```

#### 4. 审计（Audit）
- **审计日志**：记录用户对数据库的所有操作。

- **审计功能**：
  - 提供多种审计查阅方式。
  - 提供审计分析和报表功能。
  - 审计日志管理功能。
  
- **审计设置**：
  - 用户级审计：对用户创建的表或视图进行审计。
  - 系统级审计：DBA设置，监测登录和权限操作。

#### 5. 数据加密
- **数据加密**：防止数据在存储和传输中失密。
- **加密方法**：
  - **存储加密**：
    - 透明存储加密：内核级加密，对用户透明，**性能较好，安全完备性较高**
    - 非透明存储加密：通过加密函数实现。
  - **传输加密**：
    - 链路加密：链路层加密，报文和报头均加密。
    - 端到端加密：发送端加密，接收端解密，只加密报文。
- **基于SSL的传输方案**：
  - 确认通信双方端点的可靠性。
  - 协商加密算法和密钥。
  - 可信数据传输。

#### 6. 其它安全性保护
- **推理控制**：防止用户通过合法查询推导出更高密级的数据。
- **数据隐私保护**：控制个人数据的隐私，涵盖数据生命周期的各个阶段。
- **联邦学习**：
  - 2016年由谷歌提出，用于解决手机终端本地更新模型问题。
  - 在保护用户隐私和数据安全的前提下，进行数据使用和机器学习建模。
  - 应用领域：输入法、教育、医疗、推荐系统等。

#### 7. 小结
- 数据库的安全性至关重要，尤其是数据共享日益加强的背景下。
- DBMS必须具备完整而有效的安全性机制，包括：
  - 用户身份鉴别。
  - 存取控制技术（自主存取控制和强制存取控制）。
  - 视图技术。
  - 审计技术。
  - 数据加密存储和加密传输。
- 角色管理简化了授权过程，提高了权限管理的灵活性和安全性。
